# SapientiaCloud-EduPivot UI 项目说明与开发规范

## 项目概述

SapientiaCloud-EduPivot UI 是一个基于 Vue 3 + TypeScript + Naive UI 构建的现代化后台管理系统前端项目。项目采用组合式 API 开发模式，支持多语言国际化、主题切换、权限管理等企业级功能。

### 项目特点

- **现代化UI**: 基于 Naive UI 组件库，提供美观的深色/浅色主题
- **国际化支持**: 支持中文/英文双语切换
- **权限管理**: 完整的 RBAC 权限控制体系
- **响应式设计**: 适配不同屏幕尺寸
- **动态背景**: Three.js 实现的云朵/星空背景动画
- **工程化**: TypeScript + ESLint + 模块化开发

## 技术栈

### 核心框架
- **Vue 3.4+**: 采用 Composition API
- **TypeScript 5.0+**: 强类型支持
- **Vite 5.0+**: 现代化构建工具

### UI & 样式
- **Naive UI**: 主要组件库
- **SCSS**: CSS 预处理器
- **Three.js**: 3D 背景动画

### 状态管理 & 路由
- **Pinia**: 现代化状态管理
- **Vue Router 4**: 官方路由
- **Vue I18n**: 国际化方案

### 工具库
- **Axios**: HTTP 客户端
- **@vueuse/core**: Vue 工具集
- **@vicons/ionicons5**: 图标库

## 开发规范

### 1. 命名规范

#### 文件创建

- **例如**创建一个**view**下的页面文件,那么你将先创建总文件夹(settings为例), 在**settings**文件夹下创建**index.vue**文件. 
- 若有分组件(仅属于此页面)则创建**components**文件夹,在其下添加对应的**vue**文件,并在**index.vue**文件中引入. 若有比较繁杂的**css**, **scss**, **sass**样式, 则创建对应**vue**文件的文件夹,将**vue**文件和**css**, **scss**, **sass**文件(应与**vue**文件同名)放入文件夹中.
- 若一个vue文件对应多个**css**等文件,则创建**style**文件夹, 将**css**等文件放入,并创建**index.scss**(若有别的需求,也可创建其他类型文件,如**css**, **sass**, **js**(**three.js**))文件.
- 一般情况下只使用**vue**, **scss**, **ts**文件

#### 变量命名
- **变量/函数**: camelCase (如: `userName`, `getUserInfo`)
- **常量**: UPPER_SNAKE_CASE (如: `API_BASE_URL`)
- **类型/接口**: PascalCase (如: `UserInfo`, `ApiResponse`)
- **组件**: PascalCase (如: `<UserCard />`)

})

## 开发流程规范

### 1. 新建页面流程

1. **创建类型定义**: 在 `@/types` 中定义相关类型
2. **创建 API 接口**: 在 `@/api` 中定义接口和默认 DTO 函数
3. **创建页面组件**: 在 `@/views` 中创建页面
4. **添加路由配置**: 在 `@/router` 中配置路由
5. **添加国际化**: 在 `@/i18n/locales` 中添加翻译文件

### 2. 新建组件流程

1. **组件设计**: 确定组件 API 和功能
2. **类型定义**: 定义 Props 和 Emits 类型
3. **组件实现**: 编写组件逻辑和模板
4. **样式编写**: 使用 `@use` 导入样式
5. **文档编写**: 添加组件使用文档

### 3. API 开发流程

1. **类型定义**: 在 `@/types` 中定义 VO 和 DTO
2. **默认函数**: 创建 DTO 初始化函数，变量初始化值全用 null
3. **API 函数**: 实现具体的 API 调用函数
4. **错误处理**: 统一在 HTTP 工具中处理
